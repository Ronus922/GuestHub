# תוכנית GuestHub — מדריך הפעלה (עברית)

## מה יש בתיקייה הזו

| קובץ | תפקיד |
|---|---|
| `GUESTHUB_PROGRAM_V2.md` | המפרט הקנוני המלא — מקור האמת היחיד למה שצריך להיבנות |
| `00_COMMON_CHARTER.md` | האמנה: פרוטוקול השלבים, שערי כניסה/יציאה, קובץ STATE, וטבלת כיסוי שממפה כל סעיף לשלב |
| `01`–`07` | שבעה מסמכי שלבים מצטברים |

## איך זה עובד

התוכנית מחולקת ל-7 שלבים שרצים **בסדר קבוע, בלי דילוגים**. כל שלב:

1. נפתח ב"שער כניסה" שמאמת שהשלב הקודם באמת הסתיים (תגית git, בדיקות ירוקות, בטיחות סביבה מחדש).
2. מבצע רק את מה שמוקצה לו בטבלת הכיסוי שבאמנה — שום סעיף לא יכול "ליפול בין הכיסאות", כי כל סעיף ב-V2 ממופה לשלב בעלים.
3. נסגר ב"שער יציאה": בדיקות, רגרסיה, דוח שלב עם תקציר בעברית, עדכון `STATE.md`, תגית `stage-N-complete`, ועדכון ה-draft PR.

הזיכרון בין שלבים נשמר בקבצים בריפו (`docs/program/STATE.md` + דוחות השלבים), כך שאפשר להריץ כל שלב בסשן נפרד ונקי של Claude Code.

## סדר השלבים

1. **Audit וארכיטקטורה** — קריאה בלבד: גיבוי מאומת, מיפוי מלא של המערכת, מטריצת ליקויים, ארכיטקטורת יעד ו-ADRs. אפס שינויי התנהגות.
2. **Database ייעודי** — הקמת סביבת certification/staging נפרדת, roles, גיבויים אוטומטיים, והכנת cutover ל-production (בלי לבצע). התשתית המשותפת לא נגעת.
3. **שלמות הליבה** — הזמנות, מלאי, תמחור, תשלומים: מניעת double booking ברמת ה-database, מנוע מחירים קנוני, ledger, תשתית jobs עמידה, outbox לסנכרון.
4. **Channex ו-certification** — הפרדת סביבות, ledger ראיות, Full Sync מדויק, Group Update, rate limits, הזמנות נכנסות + ACK, שער הגנה ל-Production (בנוי, כבוי).
5. **השלמת יכולות PMS** — תקשורת, משק בית, תחזוקה, משימות, דוחות, ומוכנות לשוק הישראלי (מע"מ תיירים, חשבוניות, RTL).
6. **אבטחה, ביצועים, תצפיתיות** — red team מלא, בדיקות עומס, ניטור והתראות.
7. **אימות סופי ומסירה** — מאמת בלתי-תלוי, חזרה גנרלית ל-screenshare, רגרסיה מלאה, דוח סופי, PR מוכן.

## איך מריצים כל שלב

1. ודא שכל הקבצים מהתיקייה הזו נמצאים בריפו תחת `docs/program/` (שלב 1 עושה commit לזה בעצמו — פשוט העתק את התיקייה לריפו לפני ההרצה הראשונה).
2. פתח סשן חדש של Claude Code בתיקיית הפרויקט והדבק:

```
Read docs/program/00_COMMON_CHARTER.md, docs/program/GUESTHUB_PROGRAM_V2.md,
docs/program/0N_STAGE_N_....md and docs/program/STATE.md.
Execute Stage N in full, following the charter entry gate first.
```

(החלף `N` ואת שם הקובץ בשלב הרלוונטי.)

3. בסיום השלב: קרא את `docs/program/reports/STAGE_N_REPORT.md` (יש בו תקציר בעברית), עבור על מפת ה-review ב-PR, ואם הכל תקין — הפעל את השלב הבא.

## מה נשאר רק לך (אחרי שלב 7)

* לעבור על ה-draft PR ולעשות merge.
* למלא את טופס ה-certification הרשמי של Channex עם ה-Task IDs והתשובות המוכנות.
* לקבוע ולבצע את שיחת ה-screenshare החיה — לפי התסריט המוכן ב-`docs/channex/SCREENSHARE_DEMO_SCRIPT.md`.
* לאשר בעתיד את ה-cutover של ה-database וההפעלה ב-Production.

## כללי ברזל (תקציר סעיף 3 ב-V2)

בשום שלב: אין נגיעה ב-production ‏(DB, PM2, ערוצים חיים), אין merge ל-main, אין deploy, אין הפעלת Channex Production, אין commit של סודות, והתשתית המשותפת עם הפרויקטים האחרים נשארת בדיוק כמו שהיא.
