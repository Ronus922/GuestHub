# GuestHub — Full Codebase Audit (Agent B)

- **Date:** 2026-07-18
- **Branch:** `feat/pms-hardening-channex-certification`
- **Scope:** entire repository source — `src/` (app routes, server actions, components, lib/domain modules), `scripts/`, `db/migrations/` — covering reservations, calendar, rooms, rates, payments, messaging, communications, settings, auth, channel (Channex) integration and the PM2 worker.
- **Mode:** READ-ONLY. No code was modified; no servers, migrations or DB writes were touched.

## Methodology

1. **Tree + size census:** enumerated all 365 TS/TSX/MJS/SQL source files (~77k lines); ranked by line count to find decomposition candidates.
2. **Full reads** of the core domain spine: `lib/channel/worker.ts`, `lib/channel/queue.ts`, `lib/channel/booking-import.ts`, `lib/pricing/engine.ts`, `lib/pricing/reservation-pricing.ts`, `lib/payments/{ledger,gateway,collection}.ts`, `lib/auth/{actor,guards}.ts`, `lib/{db,audit,inventory,dates,card-vault}.ts`, `lib/realtime/{events,hub,publish}.ts`, `lib/communications/{worker,delivery}.ts`, `reservations/actions.ts` + `card-actions.ts` + `data.ts`, `calendar/{data,actions}.ts`, `rooms/actions.ts` (partial), `staff/actions.ts` (partial), `settings/actions.ts`, `login/actions.ts`, `middleware.ts`, all `src/app/api/**` routes.
3. **Systematic greps** across `src/`, `scripts/`, `db/`: TODO/FIXME/HACK/placeholder/stub; `: any`/`as any`; `console.log`; empty/swallowing `catch`; `.catch(() => {})`; floating-point money (`parseFloat`, `toFixed`, `* 100 / 100`); `toLocaleString` without locale/timeZone; `new Date()` on date-only strings; `JSON.parse` without guard; `sql.unsafe`; `dangerouslySetInnerHTML`; `Math.random`; `process.env` non-null assertions; `parseInt` without radix.
4. **Circular-dependency analysis:** `madge --circular` over 283 modules.
5. **Auth coverage check:** every server-action file verified to call `getActor` + `requirePermission` (only `login/actions.ts` is legitimately unauthenticated).

**Baseline hygiene (verified clean):** `console.log` in product code — **0** occurrences. `any` types — **0** real occurrences (2 grep hits are inside comments). Empty `catch {}` blocks — **0**; every `catch` either returns a typed failure, logs a sanitized `console.error`, or carries an explanatory comment. All 220 catch sites sampled follow the fail-closed/toast pattern. Tenant scoping (`actor.tenantId` in every query) and permission gates were consistent in every action file inspected. Money totals are summed in integer cents inside the engine; PAN storage is AES-256-GCM fail-closed; CVV is structurally absent (migration 018). This is an unusually disciplined codebase — the findings below are the real residue, not a sample.

## Findings

Severity legend: **Critical** = data loss / security hole now · **High** = wrong money/state in a real flow · **Medium** = correctness-adjacent debt, duplication, structural risk · **Low** = hygiene, latent risk, documented-but-unfinished.

| 1 | **High** | Channel import / payments | An inbound OTA **modification** overwrites `total_price` with the channel total only (`total = round2(norm.amount ?? agg.roomsTotal)`), while locally-added `discount_amount` / `extra_charges` columns are left untouched. ## Notes on what was checked and found clean

- **Unreachable code / hidden TODOs:** no `TODO`/`FIXME`/`HACK` markers exist in product code; the only "ponytail:" annotations are honest scale-deferral notes with conditions attached (all listed above where they matter).
- **Timezone/date handling:** all date-only math flows through `lib/dates.ts` (UTC-noon anchor); every `new Date(\`${d}T12:00:00Z\`)` usage follows the canonical pattern; `todayInTz` is used for "today". No naive `new Date("YYYY-MM-DD")` found.
- **Validation:** zod schemas cover reservations, rooms, rates, rate-plans, commercial, users; the exceptions are noted in findings #10 (card actions) — settings VAT uses a dedicated pure parser (acceptable).
- **Secrets:** no secrets in source; env access is centralized; card vault and channel crypto fail closed on missing keys.
